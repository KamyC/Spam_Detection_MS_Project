Service APIs

We have provided 3 APIs for registered and non-registered users to detect and report tweets. Non-registered users can get the results produced by an initial ML model. Registered users can input invoke the APIs by passing login information and tweets for detection or report

Using Our APIs to Detect and Report Spam

API Call Parameters Usage
http://127.0.0.1:5000/api/detect/public?tweets={tweet content} tweet content | The tweet content you would like to detect, applicable for non-registered users Use public model to detect an input tweet
http://127.0.0.1:5000/api/detect/user?email={email}&psw={password}&tweets={tweet content} email, password | email and password that you need to login your account
tweet content | The tweet content you would like to detect, applicable for registered users
 Login user's account and use user-specific model to detect an input tweet
http://127.0.0.1:5000/api/report/user?email={email}&psw={password}&tweets={tweet content} email, password | email and password that you need to login your account
tweet content | The tweet content you would like to report, applicable for registered users
 Login user's account and feed a tweet with confirmed label to a user-specific model
