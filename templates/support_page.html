Technical Support

We are at the initial stage of applying traditional ML training method to build a base model. We so far have tested KNN, naive bayes and logistic regression approaches to train the base model using dataset we have collected from Twitter.

ML Approach Accuracy Recall Precision
KNN 79.75% 91.06% 74.55%
Naive Bayes 90.41% 83.30% 97.30%
Logistic Regression 87.76% 90.11% 86.46%

On-going Model Maintenance

Since we are using online learning method to feed and update models. For any registered user with a user-specific model, every tweet fed to the model will help to improve the accuracy of detection result. It is noted that different users may hold varying standard regarding "spam" tweets. Therefore, it is very likely to deliver opposite labels from two different user models. Such divergence originates from users' involvement in confirming if model's label for certain tweet is correct or not. We will keep updating the base model and consider other algorithms to improve the accuracy. Moreover, we will deliver more APIs to access our ML model for any researchers who are interested in tweet spam detection.

We provide following technical supports:

  • Users can send help to Admin for any suspected tweets. Admin can help confirm the tweets' label
  • We are currently applying logistic regression to achieve the online learning approach. We can also provide more algorithmic support by integrating other machine learning models
  • More open APIs will be developed to public for any research purpose. Any registered users can call our service APIs in their own project
  • Data visualization interface is accessible for registered users by gathering the past input tweets. We can provide service to abstract and visualize given datasets of spam tweets.
